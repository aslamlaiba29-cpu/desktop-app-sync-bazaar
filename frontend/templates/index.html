Sprint Planning & Team Agile Methodology

The "Synchronized Bazar Store" application was built following the Agile/Scrum Framework. Agile was chosen over Waterfall because real-time synchronization requirements, dynamic vendor inventory levels, and complex cart state models required continuous iteration, feedback cycles, and short sprint cycles to tackle integration risk.

✨ Refactoring Examples (Before vs After)

❌ Before: Tightly Coupled Cart Calculations

def calculate_cart_totals(items):
    subtotal = 0
    # Calculations are direct and hardcoded
    for item in items:
        subtotal += item['price'] * item['qty']
    # Hardcoded tax rates and shipping rules inside API
    tax = subtotal * 0.08
    shipping = 10.0
    total = subtotal + tax + shipping
    return subtotal, tax, shipping, total

✓ After: Modularized Refactored Pattern

class CartManager:
    TAX_RATE = 0.08
    FLAT_SHIPPING = 10.0

    @classmethod
    def get_totals(cls, subtotal):
        # Configuration is encapsulated
        tax = round(subtotal * cls.TAX_RATE, 2)
        shipping = cls.FLAT_SHIPPING if subtotal > 0 else 0
        grand_total = round(subtotal + tax + shipping, 2)
        return {"subtotal": subtotal, "tax": tax, "shipping": shipping, "grand_total": grand_total}

Synchronized Bazar HTTP REST API Routes

POST /api/auth/register Registers a User session (Customer, Vendor, Admin)
POST /api/orders/checkout Submits cart, validates stock levels, decrement products, announcements
GET /api/sync/stream Server-Sent Events (SSE) stream client channels
POST /api/legacy/scan Runs static analyzer parsing python code smells
POST /api/testing/run Launches pytest subprocess parsing outputs

Deployment and Hosting Guide

Deploying on Render / Railway

  1. Create a PostgreSQL/MySQL database addon.
  2. Bind environment variables: `SECRET_KEY`, `DATABASE_URL` pointing to production database.
  3. Set Start Command: `gunicorn app:app` or run python using `python app.py --server-only` to bypass desktop window initialization.
